# 贪吃蛇游戏代码说明

这是一个使用 HTML、CSS 和原生 JavaScript 编写的简易贪吃蛇小游戏。项目不依赖第三方库，也不需要打包工具，直接在浏览器中打开 `index.html` 即可运行。

## 文件结构

```text
snake/
├── index.html   # 页面结构，包含标题、得分区、画布、按钮和操作提示
├── style.css    # 页面样式，负责布局、按钮、画布和整体视觉效果
├── game.js      # 游戏核心逻辑，负责绘制、移动、碰撞检测、计分和事件处理
└── readme.md    # 当前说明文档
```

## 运行方式

1. 进入 `snake` 文件夹。
2. 用浏览器打开 `index.html`。
3. 点击“开始游戏”按钮。
4. 使用键盘方向键控制蛇移动。
5. 点击“暂停”按钮可以暂停游戏，再次点击可以继续。

也可以在项目根目录使用本地静态服务器预览，例如：

```bash
python3 -m http.server 8000
```

然后在浏览器中访问：

```text
http://localhost:8000/snake/
```

## 页面结构说明

`index.html` 是游戏的入口文件，主要包含以下内容：

- `h1`：页面标题“贪吃蛇游戏”。
- `.score-board`：显示当前得分，分数值由 `#score` 元素承载。
- `canvas#gameCanvas`：游戏画布，宽高均为 `400px`。
- `#startBtn`：开始游戏按钮。
- `#pauseBtn`：暂停或继续按钮。
- `.instructions`：键盘操作提示。
- `<script src="game.js"></script>`：引入游戏逻辑脚本。

## 样式说明

`style.css` 负责游戏界面样式：

- 使用全局选择器清除默认边距，并启用 `box-sizing: border-box`。
- `body` 使用渐变背景，并通过 Flex 布局让游戏容器居中显示。
- `.container` 是白色游戏面板，包含阴影、圆角和内边距。
- `#gameCanvas` 是游戏区域，带边框和浅灰色背景。
- `button` 定义按钮颜色、间距、圆角、鼠标指针和悬停效果。

## 游戏核心逻辑

`game.js` 是整个项目的核心，负责维护游戏状态、绘制画面、处理输入和判断游戏结束。

### 主要常量

```js
const gridSize = 20;
const tileCount = canvas.width / gridSize;
```

- `gridSize` 表示一个格子的大小，当前为 `20px`。
- `tileCount` 表示画布一行或一列能容纳多少个格子。画布宽度是 `400px`，所以一行有 `20` 个格子。

游戏逻辑使用网格坐标，而不是直接使用像素坐标。例如蛇头位置 `{x: 10, y: 10}` 表示第 10 列、第 10 行，在绘制时会乘以 `gridSize` 转换成像素位置。

### 主要状态变量

```js
let snake = [{x: 10, y: 10}];
let food = {};
let dx = 0;
let dy = 0;
let score = 0;
let gameRunning = false;
let gamePaused = false;
let gameLoop;
```

- `snake`：蛇身体数组，数组第一个元素是蛇头。
- `food`：食物的位置。
- `dx`、`dy`：蛇每次移动的方向。
- `score`：当前得分。
- `gameRunning`：游戏是否正在运行。
- `gamePaused`：游戏是否处于暂停状态。
- `gameLoop`：保存 `setInterval` 返回值，用于结束游戏时清除循环。

## 主要函数说明

### `randomFoodPosition()`

随机生成食物位置。

函数会在画布网格范围内随机生成 `x` 和 `y` 坐标，并检查食物是否和蛇身体重合。如果重合，就重新生成。

### `drawGame()`

绘制当前游戏画面。

执行流程：

1. 清空画布，并绘制浅灰色背景。
2. 遍历 `snake` 数组，绘制蛇的每一节身体。
3. 根据 `food` 坐标绘制食物。

蛇使用绿色绘制，食物使用橙红色绘制。

### `moveSnake()`

更新蛇的位置，是游戏移动逻辑的核心。

执行流程：

1. 如果方向为 `dx = 0` 且 `dy = 0`，说明玩家还没有按方向键，蛇不移动。
2. 根据当前蛇头和移动方向计算新的蛇头位置。
3. 检查新蛇头是否撞墙。
4. 检查新蛇头是否撞到自己的身体。
5. 将新蛇头插入 `snake` 数组开头。
6. 如果吃到食物，分数增加并重新生成食物。
7. 如果没有吃到食物，移除蛇尾，让蛇保持原长度。

### `gameOver()`

处理游戏结束。

当蛇撞墙或撞到自己时会调用该函数。函数会停止游戏循环，并弹窗显示最终得分。

### `startGame()`

开始一局新游戏。

执行流程：

1. 如果游戏已经在运行，则直接返回。
2. 重置蛇的位置、移动方向和分数。
3. 将游戏状态设置为运行中、非暂停。
4. 随机生成食物。
5. 使用 `setInterval` 每 `100ms` 执行一次游戏循环。

游戏循环中，如果当前没有暂停，会依次执行：

```js
moveSnake();
drawGame();
```

### `togglePause()`

切换暂停状态。

如果游戏没有运行，点击暂停按钮不会生效。游戏运行时，点击按钮会切换 `gamePaused`，并把按钮文字改成“继续”或“暂停”。

## 键盘控制说明

代码通过 `document.addEventListener('keydown', ...)` 监听键盘事件。

支持的按键：

- `ArrowUp`：向上移动。
- `ArrowDown`：向下移动。
- `ArrowLeft`：向左移动。
- `ArrowRight`：向右移动。

为了避免蛇直接反向移动导致立刻撞到自己，代码中做了方向限制：

- 当前向下移动时，不能立刻改为向上。
- 当前向上移动时，不能立刻改为向下。
- 当前向右移动时，不能立刻改为向左。
- 当前向左移动时，不能立刻改为向右。

## 计分规则

每吃到一个食物：

- 分数增加 `10`。
- 页面上的得分显示同步更新。
- 新食物会随机出现在空白格子中。

## 游戏结束条件

出现以下任意情况时，游戏结束：

- 蛇头移动到画布边界之外。
- 蛇头撞到蛇自己的身体。

## 可扩展方向

可以在当前代码基础上继续扩展：

- 增加难度选择，例如调整游戏循环间隔。
- 增加最高分记录，使用 `localStorage` 保存。
- 增加重新开始按钮或游戏结束遮罩层。
- 增加移动端触摸控制。
- 增加音效和更丰富的视觉效果。
- 增加暂停时的提示文本。
